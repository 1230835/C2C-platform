# C2C-platform

objectives -> Consumer-to-Consumer online marketplace
types of users -> user, admin
priviledges -> admin has the highest level of authority, capable of adding tags and removing them at all times aswell as remove products and terminate accounts, other priviledges planned. The admin is also tasked with approving/dissaproving/bargaining user requested tags in the creation of the product planned to be put on sale and dissaproving the creation of a product sale if it doesnt follow protocol, both bargaining, addition and removal of tags on the product requested to be put for sale will be followed by a notification to the user's notification box where he will be capable of checking out the changes made, followed by an accept and decline boxes where the user can click to accept, thus, being added to the website or decline, where the user can choose between giving up on the request, or do a counter-proposal. The admin should not outright shut down the request while in the bargaining stage, but can do so.
